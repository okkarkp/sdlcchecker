# Organization memory — index

> Root file the orchestrator's pre-brief reads first. Keep entries here short and link out to the
> topic file that has the detail. See `README.md` in this folder for the entry format and the
> review gate; see `docs/organization-memory.md` in the main repo for the full three-tier model.

## How to read this file (agents)

Skim every `ACTIVE` entry below and in the linked topic files; fold anything relevant to the
current feature into your pre-brief/design/review the same way you would a project convention.
Entries marked `SUPERSEDED` or `RETIRED` are history — don't act on them, but don't delete them.

## Topic files

- [`conventions.md`](conventions.md) — cross-project coding/API conventions
- [`architecture-precedents.md`](architecture-precedents.md) — reusable ADR-style precedents
- [`security-findings.md`](security-findings.md) — recurring vulnerability classes
- [`review-anti-patterns.md`](review-anti-patterns.md) — recurring standards/reachability findings
