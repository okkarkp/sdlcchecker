# Security findings (org memory)

Recurring vulnerability classes and anti-patterns, generalized past any one project's
specifics — read by `security-reviewer` and `code-reviewer`. A finding earns a place here only
after it has recurred across more than one project; a single project's one-off finding stays
in that project's `05-review.md` and its own memory.

Never record exploit-ready specifics (real endpoints, real credentials, real payloads) here —
generalize to the *class* of issue and the *pattern* that causes it, the same way OWASP
categories describe a class of vulnerability rather than one instance.

See `README.md` in this folder for the entry format and the review gate.
