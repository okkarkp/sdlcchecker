# Review anti-patterns (org memory)

Recurring standards/reachability findings worth watching for on every future review — read by
`code-reviewer`. These are process/quality anti-patterns (e.g. a class of gap the coverage
matrix keeps missing, a reachability trap that recurred across projects), distinct from
`security-findings.md` (vulnerability classes) and `conventions.md` (positive style/API rules).

See `README.md` in this folder for the entry format and the review gate.
