⚙️ Crawl Config

The crawler reuses the Browser Agent's login (deep-DOM, shadow-root aware). This app uses a SingPass/CorpPass mock — provide UIN (and UEN for CorpPass) plus a UUID. UIN only = SingPass; UIN + UEN = CorpPass. Blank fields fall back to APP_UIN / APP_UEN / TEST_UUID in .env. Leave routes blank to auto-discover from the base URL (BFS, depth 4).

🎥 Record a Module (guided — you drive the flow)

Prefer to capture a specific flow exactly? Start a recording, then navigate the module start-to-end yourself in the browser that opens. Every page you land on is captured (elements + APIs) and tagged with the module name. Uses the same login + Base URL from Crawl Config above. Click Stop Recording (or just close the browser) when you reach the end.
